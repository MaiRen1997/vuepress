# vue3相比vue2的变化

  1. 源码使用TypeScript来进行重写
    • 在Vue2.x的时候，Vue使用Flow来进行类型检测；
    • 在Vue3.x的时候，Vue的源码全部使用TypeScript来进行重构，并且Vue本身对TypeScript支持也更好了；
  2. 使用Proxy进行数据劫持
    • 在Vue2.x的时候，Vue2是使用Object.defineProperty来劫持数据的getter和setter方法的；
      • 这种方式一致存在一个缺陷就是当给对象添加或者删除属性时，是无法劫持和监听的；
      • 所以在Vue2.x的时候，不得不提供一些特殊的API，比如$set 或 $delete，事实上都是一些hack方法，也增加了开发者学习新的API的成本；
    • 而在Vue3.x开始，Vue使用Proxy来实现数据的劫持
  3. 删除了一些不必要的API
    • 移除了实例上的$on, $off 和$once；
    • 移除了一些特性：如filter、内联模板等；
  4. 包括编译方面的优化
    • 生成Block Tree、Slot编译优化、diff算法优化；
  5. 由Options API 到Composition API：
    • 在Vue2.x的时候，我们会通过Options API来描述组件对象；
      • Options API包括data、props、methods、computed、生命周期等等这些选项；
    • 存在比较大的问题是多个逻辑可能是在不同的地方：
      • 比如created中会使用某一个method来修改data的数据，代码的内聚性非常差；
    • Composition API可以将相关联的代码放到同一处进行处理，而不需要在多个Options之间寻找
  6. Hooks函数增加代码的复用性
    • 在Vue2.x的时候，我们通常通过mixins在多个组件之间共享逻辑；
      • 但是有一个很大的缺陷就是mixins也是由一大堆的Options组成的，并且多个mixins会存在命名冲突的问题；
    • 在Vue3.x中，我们可以通过Hook函数，来将一部分独立的逻辑抽取出去，并且它们还可以做到是响应式的
  7. data的定义不同
    • vue2中，data可以是个对象，也可以是个函数(一般使用函数，为了保证组件的独立性和可复用性)
    • vue3中只能是个函数
  8. vue2中 template只允许有一个根元素，vue3中可以有多个根元素

# vue的挂载

  1. vue会挂载到mount参数里的id上，一般设置id为#app，以下描述也以#app为例
  2. 挂载时，vue会将所要挂载的innerHtml清空，并将自己的template放到该dom的innerHtml
  3. 如果和id为app的dom并列的dom，vue不会进行响应式操作，但是可以当作原生html显示

# method不能使用箭头函数

  1. 箭头函数绑定了父级作用域的上下文，所以this将不会按照期望的指向组件实例，所以箭头函数的this都不能使用了
  2. 同样的，methods中的this指向的是组件的实例

# vuev中的{{}}语法

​ tips:双大括号并不会解析html标签

  1. {{}}中可以输入变量
  2. {{}}中可以输入methods中的带有返回值的函数
  3. {{}}中可以输入数组或者字符串的api进行运算
  4. {{}}中可以输入表达式
  5. {{}}中可以输入三元运算符

# 指令的用法

  1. v-once的使用

    v-once用于指定元素 / 组件只渲染一次

    使用方法

    <div v-once>
    {{count}}
</div>
    • 当数据发生变化时， 元素或者组件及其所有子元素将视为静态内容并且跳过
    • 该指令可以用于性能优化

  2. v-text的使用

    用于更新元素的textContent

    <div v-text="textInfo"></div>
<!--等价于-->
<div>{{textInfo}}</div>

    但是相对于 {{}}，功能比较单一推荐使用{{}}语法

  3. v-html语法

    如果我们希望网页标签可以被Vue可以解析出来，那么可以使用v-html 来展示

    <div v-html="textInfo"></div>
<!--等价于-->
<div>{{textInfo}}</div>

    // 定义textInfo
textInfo = `<div>这是第{{count}}次测试</div>`

  4. v-pre

    用于跳过元素和它的子元素的编译过程，显示原始的{{}}标签

    也就是不解析{{}}内的变量

  5. v-cloak

    这个指令保持在元素上直到关联组件实例结束编译

    和CSS 规则如[v-cloak] { display: none } 一起用时，这个指令可以隐藏未编译的Mustache 标签直到组件实 例准备完毕。

    <div v-cloak>
    {{message}}
</div>

    <style>
[v-cloak] {
  display: none;
}
</style>

    上面的代码可以保证div标签不会显示，直到编译结束

  6. v-bind

    1. 动态绑定属性，可以简写为 冒号:

      <img v-bind:src="srcUrl" />
<!--等价于-->
<img :src="srcUrl" />

      动态绑定一个或者多个属性， 或者传递props

# 动态绑定class

  1. 对象语法

    1. 方式1 直接绑定大括号对象

      <div :class="{ active: isActive }"></div>
<!--isActive为布尔值，可以定义在data中，通过methods进行更改，从而达到动态样式效果-->

    2. 方式2， 直接绑定大括号，并添加固定不变的样式

      <div
   class="static"
   :class="{ active: isActive,'text-danger': hasError }"
 ></div>
<!--
isActive为布尔值，可以定义在data中，通过methods进行更改，从而达到动态样式效果
static为标签默认的样式，或者说是不随着状态改变的公共样式
-->

      tips： 对象里的变量名，可以加引号也可以不加，如果是用连字符的样式名，最好要加引号

    3. 方式3， 可以把对象放到一个单独的对象里

      <div
   class="static"
   :class="classObject"
 ></div>

      setup() {
    const classObject = reactive({
      active: true,
      'text-danger': false
    })
  }

      也可以将返回的对象放到methods或者计算属性中

      methods适用于vue2

      <div
   class="static"
   :class="classFun()"
 ></div>

      // 下面的布尔值，可以在方法中定义变量，并通过变量进行控制
classFun() {
    return {
       active: true,
      'text-danger': false 
    }
}

      计算属性如下

      <div
   class="static"
   :class="classObject"
 ></div>

      setup() {
    const isActive = ref(true)
    const error = ref(null)
    const classObject = computed(() => ({
      active: isActive.value && !error.value,
      'text-danger': error.value && error.value.type === 'fatal'
    }))
  }

  2. 数组语法

    1. 普通用法

      <div :class="[activeClass, errorClass]"></div>

      setup() {
    const activeClass = ref('active')
    const errorClass = ref('text-danger')
}

    2. 也可以使用三元运算符进行有条件的显示

      <div :class="[isActive ? activeClass : '', errorClass]"></div>

    3. 也可以使用数组嵌套对象的方式

      <div :class="[{ active: isActive }, errorClass]"></div>

# 动态绑定style

  1. 普通用法,直接行内使用对象

    <div :style="{ color: activeColor, fontSize: fontSize + 'px' }"></div>

    setup() {
    const activeColor = ref('red')
	const fontSize = ref(30)
}

    tips：如果不喜欢使用驼峰，可以使用引号+短横线，如

    <div :style="{ color: activeColor, 'font-size': fontSize + 'px' }"></div>

  2. 可以使用对象的模板

    setup() {
    const styleObject = reactive({
      color: 'red',
      fontSize: '13px'
    })
 }

    <div :style="styleObject"></div>

  3. 使用style的数组对象

    setup() {
    const styleObject = reactive({
      color: 'red',
      fontSize: '13px'
    })
    const styleObject2 = reactive({
      fontWeight: 500,
      background: 'red'
    })
 }

    <div :style="[styleObject, styleObject2]"></div>

# 动态绑定属性名

如果属性名称不是固定的，我们可以使用:[属性名]=“值” 的格式来定义

setup() {
    const name = ref('testName')
    const value = ref('123')
 }

<div :[name]="value">
    测试div
</div>
<!-- 最后渲染效果如下-->
<div testName="123">
    测试div
</div>

# 属性绑定一个对象

setup() {
    const info = reactive({
        name: 'why',
        age: 18,
        height: 1.88
    })
 }

<div v-bind="info">测试div</div>

tips：直接使用v-bind,什么属性都不用加，这样对象里的键值对，就会通过 键值对的形式渲染到div上， 如 name = why这样的形式
